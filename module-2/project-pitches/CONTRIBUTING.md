# Contributing

## Our Values

We build with compassion. Reviews should be constructive, kind, and focused on helping each other think through ideas. No pitch is too ambitious or too simple. If someone wants to build a global supply chain tracker and someone else wants to build a local sanctuary volunteer scheduler, both are valuable contributions to the movement.

## Branch Naming

For this exercise, use the format: `pitch/short-description`

Examples:
- `pitch/vegan-product-scanner`
- `pitch/shelter-network-api`
- `pitch/factory-farm-emissions-map`
- `pitch/advocacy-chatbot`

Keep it lowercase, use hyphens, and make it descriptive enough that someone can guess what the pitch is about from the branch name.

## Good Commit Messages

Format: `Add pitch: [project name]`

**Good:**
- `Add pitch: sanctuary supply tracker`
- `Update pitch: clarify target users for chatbot`
- `Fix pitch: correct emissions statistic source`

**Bad:**
- `update` (update what?)
- `added my file` (what file? what's in it?)
- `asdfgh` (no)
- `final version` (it won't be)

A good commit message tells someone what changed without them having to open the file. Write it as if you're leaving a note for a teammate.

## Good PR Reviews

The point of reviewing a classmate's pitch is to engage with their idea, not just rubber-stamp it. Here's what good engagement looks like:

**Ask questions:**
- "Have you seen [similar project]? It does something related — could be worth looking at for inspiration."
- "Who would maintain the data in this tool? Is that something that could be automated?"
- "What happens when two sanctuaries claim the same donation?"

**Offer encouragement:**
- "This would be huge for local groups that don't have technical resources."
- "I've never thought about this problem before — the fact that sanctuaries can't share surplus supplies is wild."

**Suggest connections:**
- "This could integrate with [other student]'s idea about volunteer coordination."
- "Open Paws already has a factory farm API — your tool could pull data from that."

**Don't** just write "Looks good" or "LGTM". If you're approving, say what specifically you think is strong about the idea.

## Merge Conflicts

If you see a merge conflict, don't panic. This happens when two people edit the same file at the same time. Ask your instructor for help — we cover conflict resolution in a later session. Do not force-push or delete your branch to "fix" it.

## File Naming

Your pitch file goes in the `pitches/` directory. Name it: `your-name-short-title.md`

Examples:
- `sam-td-shelter-coordinator.md`
- `priya-k-deforestation-monitor.md`
- `alex-r-vegan-ingredient-scanner.md`

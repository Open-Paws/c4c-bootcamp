# C4C Project Pitches — Your First Pull Request

Welcome to Code for Compassion Campus. This exercise teaches you the professional workflow every software team uses — while contributing your first idea to the movement.

## What You'll Do

You'll pitch a project idea: a tool, app, or system that could help animal advocacy, climate action, or AI safety. Then you'll submit it through the same workflow used by every open-source project in the world.

## The Workflow

1. **Fork** this repo (make your own copy)
2. **Create a branch** (make a safe workspace)
3. **Write your pitch** (describe the tool you want to build)
4. **Commit** (save your work with a clear message)
5. **Open a Pull Request** (submit for review)
6. **Review a classmate's pitch** (give constructive feedback)
7. **Merge** (your idea joins the collection)

## Step-by-Step Instructions

### 1. Fork This Repository

Click the **Fork** button in the top-right corner of this page. This creates your own copy under your GitHub account.

### 2. Create a Branch

In YOUR forked copy, click the branch dropdown (says "main"). Type a new branch name: `pitch/short-description` (e.g., `pitch/shelter-coordinator-bot`). Click "Create branch".

### 3. Write Your Pitch

Navigate to the `pitches/` folder. Click **Add file** → **Create new file**. Name it: `your-name-short-title.md` (e.g., `sam-td-shelter-coordinator.md`).

Use the template below.

### Pitch Template

Copy this into your new file and fill it in:

```
# [Project Name]

**Pitched by:** [Your name]
**Category:** [Animal Advocacy / Climate Action / AI Safety / Multiple]

## The Problem
What specific problem does this solve? Who experiences it? Why does it matter?
(2-3 sentences)

## The Tool
What would this tool actually do? How would someone use it?
(2-3 sentences)

## Who It Helps
Which organisations, advocates, or communities would use this?
(1-2 sentences)

## Tech I Think It Needs
What technologies might be involved? (It's OK to guess — that's what this course is for)
(A few words or a short list)

## Why I Care About This
What draws you to this particular problem?
(1-2 sentences)
```

### 4. Commit Your File

Scroll down to "Commit changes". Write a descriptive message: `Add pitch: [project name]`. Select "Commit directly to the `pitch/...` branch". Click **Commit changes**.

### 5. Open a Pull Request

Go back to the **original repository** (not your fork). Click **Pull Requests** → **New Pull Request**. Click "compare across forks". Set base: original/main ← head: your-fork/your-branch. Click **Create Pull Request**. The PR template will appear — fill it in and submit.

### 6. Review a Classmate's Pitch

Find someone else's open Pull Request. Click **Files changed** to read their pitch. Leave a constructive comment: What excites you? What questions do you have? Can you think of a similar tool that already exists? Click **Review changes** → **Approve** → **Submit review**.

### 7. Merge

Once approved, click the green **Merge pull request** button. Your first contribution to the movement's codebase.
